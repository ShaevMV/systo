> **Статус:** дизайн-гейт СНЯТ владельцем (2026-06-20). Готово к реализации сериями PR.
> **Контекст:** критический путь к надёжному офлайн-входу на КПП (план `baza-update-implementation-plan.md` §Ф5). Greenfield рядом со старым Blade (Strangler) — старый `/scan`,`/search`,`/enter` остаётся fallback ≥1 фестиваль, cutover после паритета и доказанного офлайна.

## Целевое устройство
Телефон (портрет), крупные кнопки, мгновенный отклик, тач. Камера требует secure context (HTTPS) → локальный SSL (mkcert/локальный CA).

## Многоагентный архитектурный разбор (2026-06-20) + решения PM за владельца

> Владелец: «вся система Baza переезжает на новые рельсы… с учётом плохого интернета, мобильных и офлайна прямо с телефона» + «решения принимает project-manager, я ему доверяю».
> Прошёл разбор 4 агентов (devops + архитектор + frontend + security) → синтез ведущего архитектора. PM принял 15 решений за владельца. Полный синтез — в транскрипте сессии (workflow `baza-new-rails-architecture`).

### Рекомендуемая архитектура (итог)
- **Переезжает ТОЛЬКО UI (Blade→PWA).** Laravel-Baza остаётся **API-ядром** (уже Laravel 11 / Sanctum 4, чистая CQRS, БД только в репозиториях — UI это деталь по Dependency Rule). Бэкенд НЕ переписываем, только **аддитивно расширяем**.
- **Истина данных при офлайне = локальный узел КПП** (ноутбук/мини-сервер в палатке), один на КПП. Телефон — **тонкий клиент**, но с автономией: держит (а) кэш оболочки (service worker), (б) **минимизированный шифрованный снимок билетов** в IndexedDB (только поля впуска), (в) **append-only очередь намерений впуска**.
- **Auth: онлайн-вход = session-куки Baza + CSRF** (НЕ JWT); **офлайн-вход = PIN** (Argon2id, разблокирует AES-GCM ключ кэша). ⚠️ Прежняя формулировка «session-auth через куки офлайн» — техошибка (кука валидируется сервером, офлайн бесполезна).
- **Синк двухуровневый:** телефон↔узел КПП (онлайн по локальному Wi-Fi; обрыв → дренаж клиентской IndexedDB-очереди `POST /api/enter` по `client_op_id`); узел↔боевой сервер (ingest Ф3 + webhook-outbox Ф4 + sneakernet NDJSON-ZIP как аварийный канал).
- **Отметки впуска — append-only** (НЕ last-write-wins по строке билета; `ImportApplication::isFresher` = LWW по `updated_at` остаётся только для каталога билетов, который мастерит org).
- **Стратегия — Strangler (единогласно 4/4), big-bang отвергнут.** Старый Blade — fallback ≥1 фестиваль; cutover только после паритета + доказанного офлайна на реальном железе.

## Архитектура приложения
- **Отдельное приложение** `BazaFront/` (Vite + Vue 3 + PrimeVue, по образцу `AdminFront/`, но БЕЗ JWT-обвязки — `axios.withCredentials=true` + `X-XSRF-TOKEN`, session-куки Baza).
- **Docker-сервис** `node-baza`, раздача под `/baza/` (как `AdminFront` под `/admin/`).
- **Offline-first:** service worker (Workbox, `registerType:'prompt'` — НЕ autoUpdate) + IndexedDB (минимизированный шифрованный снимок + очередь намерений впуска).
- **CI-job сборки** ЗАРАНЕЕ (закрывает TD-32 первым PR).
- Старый Blade (`/scan`,`/search`,`/enter`) — fallback, не трогаем до cutover.
