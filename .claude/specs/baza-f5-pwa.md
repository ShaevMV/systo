> **Статус:** дизайн-гейт СНЯТ владельцем (2026-06-20). Готово к реализации сериями PR.
> **Контекст:** критический путь к надёжному офлайн-входу на КПП (план `baza-update-implementation-plan.md` §Ф5). Greenfield рядом со старым Blade (Strangler) — старый `/scan`,`/search`,`/enter` остаётся fallback ≥1 фестиваль, cutover после паритета и доказанного офлайна.

| Вопрос | Решение |
|--------|---------|
| **Главный экран** | **Сразу сканер (камера)**. Шапка: текущий КПП/смена. Снизу таб-бар: Скан / Поиск / Смена. Под камерой — «ввести № вручную». Максимум скорости на входе. |

## Целевое устройство
Телефон (портрет), крупные кнопки, мгновенный отклик, тач. Камера требует secure context (HTTPS) → локальный SSL (mkcert/локальный CA).

### Рекомендуемая архитектура (итог — облако-мастер, пересинхрон 2026-06-21)

> **⚠️ Топология изменена решением владельца (встреча 2026-06-21, реш. #13=A): «облако-мастер, ноут опционален».** Раньше спека была под «узел-мастер на КПП» — ниже актуальная редакция. Узел КПП оставлен строго как **аварийный кэш на блэкаут**, а не как обязательная истина.

- **Переезжает ТОЛЬКО UI (Blade→PWA).** Laravel-Baza остаётся **API-ядром** (уже Laravel 11 / Sanctum 4, чистая CQRS, БД только в репозиториях — UI это деталь по Dependency Rule). Бэкенд НЕ переписываем, только **аддитивно расширяем**.
- **Истина данных = облако (боевой сервер Baza).** Телефон — **тонкий клиент** с автономией на случай обрыва связи: держит (а) кэш оболочки (service worker), (б) **минимизированный шифрованный снимок билетов** в IndexedDB (только поля впуска, B5), (в) **append-only очередь намерений впуска** (outbox).
- **Ноутбук/мини-сервер на КПП — ОПЦИОНАЛЕН** (аварийный кэш-реле на случай полного блэкаута сотовой), НЕ обязателен и НЕ является истиной. По умолчанию КПП работает без ноута: телефон ↔ облако напрямую.
- **Auth: онлайн-вход = session-куки Baza + CSRF** (НЕ JWT); **офлайн-вход = PIN** (Argon2id, разблокирует AES-GCM ключ кэша). ⚠️ Прежняя формулировка «session-auth через куки офлайн» — техошибка (кука валидируется сервером, офлайн бесполезна).
- **Синк ОДНОУРОВНЕВЫЙ: телефон ↔ облако** (онлайн по сотовой/локальному Wi-Fi; обрыв → дренаж клиентской IndexedDB-очереди `POST /api/enter` по `client_op_id`/`nonce` при восстановлении связи). Снимок (`GET /api/snapshot`) и журнал проходов (`baza_entry_events`) живут **на боевом сервере**, не на узле. Узел-реле (если поднят) лишь проксирует/кэширует тот же контракт — отдельный кодовый путь синка НЕ городим.
- **Отметки впуска — append-only** (НЕ last-write-wins по строке билета; `ImportApplication::isFresher` = LWW по `updated_at` остаётся только для каталога билетов, который мастерит org).
- **Стратегия — Strangler (единогласно 4/4), big-bang отвергнут.** Старый Blade — fallback ≥1 фестиваль; cutover только после паритета + доказанного офлайна на реальном железе.

## Архитектура приложения
- **Отдельное приложение** `BazaFront/` (Vite + Vue 3 + PrimeVue, по образцу `AdminFront/`, но БЕЗ JWT-обвязки — `axios.withCredentials=true` + `X-XSRF-TOKEN`, session-куки Baza).
- **Docker-сервис** `node-baza`, раздача под `/baza/` (как `AdminFront` под `/admin/`).
- **Offline-first:** service worker (Workbox, `registerType:'prompt'` — НЕ autoUpdate) + IndexedDB (минимизированный шифрованный снимок + очередь намерений впуска).
- **CI-job сборки** ЗАРАНЕЕ (закрывает TD-32 первым PR).
- Старый Blade (`/scan`,`/search`,`/enter`) — fallback, не трогаем до cutover.

**PR-1 + PR-2 — done (PWA-каркас + service worker).** 

**Жёсткий порядок офлайн-входа (реш. встречи #14=A, 2026-06-21):**
`PR-3 (snapshot) → PR-6 (blacklist отозванных + шифрование) → PR-4 (офлайн-впуск)`. **Офлайн-впуск (PR-4) НЕ выходит на боевой КПП, пока не слит PR-6** — иначе отменённый/возвращённый билет пройдёт офлайн (закрывает топ-риск «отзыв не доходит до КПП»). Сам экран сканера можно разрабатывать параллельно, но **боевой офлайн-режим включается только после PR-6**.

**До PR-8 (append-only журнал) КПП работает строго на ОДНОМ устройстве** — нет журнала проходов → два телефона могут пустить один билет дважды. PR-8 = 🚪 **жёсткий гейт мульти-устройства**.

PR-7/9 ждут факты феста (A1/D12/D14). PR-9 теперь опционален (облако-мастер).
