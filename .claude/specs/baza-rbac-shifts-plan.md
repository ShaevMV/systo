## Корень бага поиска «test» (РАЗОБРАН)
- **Эмпирика (БД стенда):** «test» вернул демо-записи, потому что у них **email с `test`**
  (`sc1-ivan@test.com`, `driver1@spaceofjoy.test`) → поиск по email справедливо их находит.
  На РЕАЛЬНЫХ данных гостей «test» так не выстрелит. Числовой баг тут НЕ сработал (0 совпадений).
- **Латентный баг (по коду):** legacy `find()` делали `whereKilter((int)$q)` и
  `auto LIKE '%'.(int)$q.'%'`. Для нечислового `(int)"test" === 0` → поиск по kilter=0 /
  номеру с нулём = «шляпа» на других данных. Эталон без бага — `InMemoryMySqlTicketSearch`
  (числовой поиск только при `ctype_digit`).
- **Фикс (Шаг 1, сделан):** в 5 legacy-репо — числовой поиск только при `ctype_digit($q)`,
  парковка по госномеру текстом (без `(int)`), `comment` убран из гостевого поиска.
  Демо-почту с `test` почистить в сидерах (Шаг 2). Тест `SearchRelevanceTest`.
