## I. Дополнительные опции к билетам

### Бизнес-суть

Вводится новое понятие — **«опция»**: дополнительное условие, прикрепляемое к основному билету. Опция — отдельная сущность со своей стоимостью.

**Пример:** билет «Цифровой оргвзнос соорганизатора / 9400 руб.» можно дополнить опцией «Саженец» (за дополнительную плату). Конечный билет = тип + опции.

### Что нужно реализовать

- Новая сущность `Option` (или `TicketOption`) с полями: `id`, `name`, `price`, `ticket_type_id` (привязка)
- Админ-панель: CRUD опций + привязка к типам билетов
- Каждая опция привязывается к одному (или нескольким?) `ticket_type` — **уточнить**

### Изменения формата заказа (BREAKING CHANGE)

**Сейчас (v2.5.x):**
- `ticket_type_id` — на уровне всего заказа (одно значение)
- `promo_code` — на уровне всего заказа (один промокод)
- `guests[]` — массив с `value`, `email`

**После (v2.6.0+):**
- `ticket_type_id` — **на уровне каждого гостя** (можно смешивать в одном заказе)
- `promo_code` — **на уровне каждого гостя**
- `options[]` — массив UUID опций для конкретного гостя

**Пример нового формата:**
```json
{
  "guests": [
    {
      "id": "82ac03b3-974d-43fb-91b4-da9257090345",
      "email": "spacetransferp2p@gmail.com",
      "value": "Поэт Вася",
      "ticket_type_id": "222abc0c-fc8e-4a1d-a4b0-d345cafada08",
      "options": ["uuid_options"],
      "promo_code": "TMBR5X"
    }
  ]
}
```

### Расчёт стоимости

```
TotalPrice = Σ (по каждому гостю):
    BasePrice(ticket_type_id, current_wave)
  + Σ Option.price (выбранные опции)
  - PromoDiscount(promo_code, ticket_type_id)
```

## Подтверждённые ответы (2026-05-30 после короткого Q&A)

1. **Опции к билетам — ticket_type разный у каждого гостя.** В одном заказе можно смешивать разные типы билетов: гость1 — оргвзнос + саженец, гость2 — детский, гость3 — парковка. **Это серьёзный BREAKING CHANGE** текущей модели Order.

2. **Права на промокоды — не для роли `guest`, а для всех остальных.** То есть создавать/редактировать промокоды смогут: `admin`, `manager`, `pusher`, `curator`, `seller`, `pusher_curator`. Только обычный `guest` остаётся без этого права.

3. **qr.spaceofjoy.ru — наш сервис партнёра / внутренний.** Можем менять обе стороны. Это упрощает интеграцию — обе стороны можно эволюционировать синхронно.
