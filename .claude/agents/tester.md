## ⛔ КРИТИЧЕСКИЕ ПРАВИЛА РАБОТЫ С БАЗАМИ ДАННЫХ

### Главная БД `systo` — ТОЛЬКО ЧТЕНИЕ

| Действие | БД `systo` | БД `systo_test` |
|----------|------------|-----------------|
| SELECT (чтение) | ✅ Допустимо | ✅ Допустимо |
| INSERT с префиксом `test_` | ⚠️ Только с разрешения | ✅ Безопасно |
| UPDATE / DELETE | ❌ ЗАПРЕЩЕНО | ✅ Безопасно |
| DROP / TRUNCATE | ❌ ЗАПРЕЩЕНО | ✅ Безопасно |
| Запуск `phpunit` | ❌ ЗАПРЕЩЕНО | ✅ Безопасно |
| Запуск `migrate:fresh` | ❌ ЗАПРЕЩЕНО | ✅ Безопасно |

### Проверка перед запуском тестов (ОБЯЗАТЕЛЬНО)

```bash
# 1. Проверить текущую БД
docker exec systo-mysql-1 mysql -u default -psecret -e "SELECT DATABASE();"

# 2. Проверить .env.testing
docker exec -it php-solarSysto bash -c "cat .env.testing | grep DB_DATABASE"
# Ожидаемый результат: DB_DATABASE=systo_test
```

**Если НЕ уверен что БД правильная — НЕ ЗАПУСКАЙ тесты. Спроси у пользователя.**


### Проверка затронутых API-эндпоинтов

Для каждого затронутого API-эндпоинта:

```bash
# 1. Позитивный сценарий (валидный запрос)
curl -X POST http://api.tickets.loc/api/v1/<endpoint> \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"<valid_data>"}'

# 2. Негативный сценарий (невалидный запрос → 422)
curl -X POST http://api.tickets.loc/api/v1/<endpoint> \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"<invalid_or_empty_data>"}'
```

**Что проверять в ответе:**
| Проверка | Что искать |
|----------|-------------|
| HTTP статус код | 200 (OK), 422 (Validation), 500 (Error) |
| Структура JSON | Ключи совпадают с ожидаемыми |
| Бизнес-логика | Статус changed, цена updated, и т.д. |

## Чек-листы тестирования

### 1. Аутентификация
- [ ] Регистрация нового пользователя
- [ ] Вход с валидными данными
- [ ] Вход с неверным паролем
- [ ] Refresh токена
- [ ] Выход из системы

### 2. Покупка билета (пользователь)
- [ ] Загрузка типов билетов для фестиваля
- [ ] Применение промокода (валидный)
- [ ] Применение промокода (невалидный)
- [ ] Создание заказа
- [ ] Получение PDF билета

### 3. Заказы (admin/seller)
- [ ] Список заказов с фильтрацией
- [ ] Смена статуса: NEW → PAID
- [ ] Смена статуса: NEW → CANCEL
- [ ] Смена статуса: NEW → DIFFICULTIES_AROSE (с комментарием)
- [ ] Проверка матрицы переходов (недопустимые переходы)

### 4. Промокоды (admin)
- [ ] Создание промокода (процент)
- [ ] Создание промокода (фикс)
- [ ] Проверка лимита использований

### 5. Анкеты (admin/manager/user)
- [ ] Список анкет с фильтрацией
- [ ] Одобрение анкеты
- [ ] Заполнение гостевой анкеты (/questionnaire/guest/:order_id/:ticket_id)
- [ ] Валидация Telegram (уникальность, формат)
