# Аварийный локальный узел КПП (Ф5, PR-7 + PR-9) — ОПЦИОНАЛЬНО

> **Топология «облако-мастер» (реш. встречи 2026-06-21, #13=A):** по умолчанию телефоны
> работают против облака (`vhod.spaceofjoy.ru`) по своей **сотовой HTTPS** — локальный узел
> и этот каталог **НЕ нужны**. Всё здесь — на случай **полного блэкаута сотовой** на КПП:
> поднять автономный кэш-реле, к которому телефоны цепляются по локальному Wi-Fi.

## Что это

Узел отдаёт тот же контракт, что облако:
`/api/snapshot`, `/api/blacklist`, `/api/scan`, `/api/enter`, `/api/entry-events` + статику
BazaFront по локальному HTTPS. Телефоны синкают снимок/blacklist и дренят журнал проходов
в узел; узел позже доезжает до облака (sneakernet/любой канал) — мёрж append-only (PR-8).

## Состав каталога

| Файл | Назначение |
|------|------------|
| `setup-local-ssl.sh` (PR-7) | mkcert: локальный CA + серт на `kpp.local`/фикс-IP → `./certs` |
| `INSTALL_CA_RUNBOOK.md` (PR-7) | раскатка корневого CA на телефоны (Android/iOS) |
| `docker-compose.kpp.yml` (PR-9) | минимальный узел: mysql + php-fpm + nginx + dist |
| `build-offline-bundle.sh` (PR-9) | собрать офлайн-бандл (образы + dist + дамп БД) в `./bundle` |
| `nginx.kpp.conf` (PR-9) | конфиг nginx узла (HTTPS + проксирование API + статика `/baza/`) |

## Порядок поднятия узла (на КПП, без интернета)

1. **Заранее (со связью)** собрать бандл: `./build-offline-bundle.sh` → `./bundle/` +
   образы (`docker save`). Перенести каталог `infra/kpp` на ноут узла.
2. На ноуте: `./setup-local-ssl.sh kpp.local <IP>` → серт в `./certs`, CA в `rootCA.pem`.
3. Раскатать `rootCA.pem` на телефоны (см. `INSTALL_CA_RUNBOOK.md`).
4. `docker compose -f docker-compose.kpp.yml up -d` (предварительно `docker load` образов из бандла).
5. Телефоны: открыть `https://kpp.local/baza/` (или фикс-IP), задать PIN, синкнуть снимок.

## ⚠️ Статус (скелет — доводка на реальном железе)

Это **каркас**, не оттестированный на боевом ноуте КПП. До активации нужно (зависит от
фактов феста A1/D12/D14 — см. `.claude/specs/baza-f5-pwa.md`):
- собрать боевые образы `kpp-baza-php:offline` из `Docker/baza` (`build-offline-bundle.sh`);
- определить минимизированный дамп БД (B5 — без лишних ПДн) для `bundle/baza-dump.sql`;
- проверить swap/память ноута (≥8 ГБ), сеть, авто-старт compose при включении ноута;
- решить, кто раскатывает CA (D14).

Под облако-мастер этот узел — **запасной вариант**, поэтому его доводка не блокирует
основной офлайн-сценарий (телефон ↔ облако по сотовой).
