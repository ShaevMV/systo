# Раскатка корневого CA на телефоны КПП (Ф5, PR-7)

> **Когда это нужно:** ТОЛЬКО для аварийного локального узла КПП на **полном блэкауте**
> сотовой (топология облако-мастер — обычно телефоны ходят к облаку по своей сотовой
> HTTPS, и локальный CA не нужен вовсе). Камера PWA требует HTTPS (secure context),
> поэтому при работе телефона против локального узла по Wi-Fi узлу нужен валидный для
> телефона сертификат → на телефон надо поставить корневой CA, которым он подписан.

## 0. Предусловие
На узле КПП выполнен `infra/kpp/setup-local-ssl.sh` → есть `infra/kpp/certs/rootCA.pem`
(корневой CA) + `kpp.crt`/`kpp.key` (серт узла на `kpp.local` / фикс-IP).

Файл `rootCA.pem` нужно передать на каждый телефон (раздать по локальному Wi-Fi:
http-страничка узла, QR со ссылкой на файл, AirDrop/Bluetooth, или кабелем).

## 1. Android
1. Скопировать `rootCA.pem` на телефон (переименовать в `rootCA.crt` — некоторые версии
   Android требуют расширение `.crt`).
2. **Настройки → Безопасность → Шифрование и учётные данные → Установить сертификат →
   Сертификат CA** (путь зависит от прошивки/версии; на чистом Android:
   *Настройки → Безопасность → Дополнительно → Шифрование и учётные данные*).
3. Выбрать файл → подтвердить «Установить». Появится предупреждение «сеть может
   отслеживаться» — это норма для локального CA.
4. Проверка: открыть `https://kpp.local/` (или фикс-IP) в браузере → замок без ошибок,
   камера в PWA запускается.

> ⚠️ Android 7+ по умолчанию НЕ доверяет пользовательским CA для приложений, но **браузер
> (Chrome) — доверяет**. PWA работает в браузере → user-CA достаточно. Root/MDM не нужен.

## 2. iOS / iPadOS
1. Передать `rootCA.pem` на телефон (AirDrop / письмо / страничка узла). Safari предложит
   «Загрузить профиль».
2. **Настройки → Профиль загружен → Установить** (ввести пасскод).
3. **ВАЖНО (iOS обязательный 2-й шаг):** Настройки → Основные → **Об этом устройстве →
   Доверие сертификатам** → включить тумблер для установленного корневого CA.
4. Проверка: `https://kpp.local/` в Safari → без ошибок, камера в PWA работает.

## 3. Кто и когда раскатывает (открытый вопрос D14)
- **Ответственный:** определить до феста (тех-координатор смены/штаб).
- **Когда:** при первичной настройке телефона на смену (вместе с установкой PWA на
  главный экран). На каждый личный телефон — один раз; CA живёт до отзыва/переустановки.
- **Срок жизни CA:** mkcert root CA ~10 лет, но удобнее перевыпускать **каждый фест**
  (свежий CA + раздать) — меньше риск утечки. Старый CA удалить с телефонов после феста.

## 4. Отзыв
Снять доверие/удалить CA: Android — *Настройки → Безопасность → Учётные данные
пользователя* → удалить; iOS — удалить профиль (*Настройки → Основные → VPN и управление
устройством*). Делать после феста.

## 5. Альтернатива (если CA на телефоны не раскатать)
Топология облако-мастер это допускает: телефоны работают **против облака по своей сотовой
HTTPS** (валидный публичный серт `vhod.spaceofjoy.ru`) — тогда локальный узел/CA не нужен,
камера работает на публичном HTTPS. Локальный SSL включаем только если на КПП реально
ожидается полный блэкаут сотовой и поднимается узел-реле (PR-9).
